# 数据库优化知识库

这是我的数据库优化知识库，用来长期沉淀学习笔记、概念卡片、问题排查思路、实验记录和案例复盘。

它不是固定课程大纲，而是一个可持续扩展的知识系统：遇到一个概念就补一张卡片，解决一个问题就写一次复盘，做过一次实验就留下数据和结论。

## 入口

- [知识地图](知识地图.md)：按主题快速定位内容。
- [主题笔记](主题笔记/目录.md)：系统化整理数据库优化相关知识。
- [问题索引](问题索引/目录.md)：用问题驱动整理排查和理解路径。
- [案例库](案例库/目录.md)：记录真实或模拟优化案例。
- [实验记录](实验记录/目录.md)：记录压测、执行计划和对比实验。
- [模板](模板/目录.md)：新增笔记、问题、案例和实验时可复制的模板。

## 适合沉淀的内容

- 概念：例如覆盖索引、回表、锁等待、执行计划、统计信息。
- 方法：例如如何分析慢 SQL、如何判断索引是否有效。
- 问题：例如为什么加了索引还是慢、为什么执行计划会变。
- 案例：一次完整的优化过程和复盘。
- 实验：不同 SQL 写法、索引组合、数据量变化下的对比结果。

```text
`-- README.md
```

## 记录习惯

- 先记录问题，再补充结论；知识库允许半成品存在。
- 每条结论尽量保留证据，例如执行计划、指标、实验结果或来源。
- 区分通用原则和具体数据库实现差异，例如 MySQL、PostgreSQL、SQL Server、Oracle。
- 不只记录“怎么做”，也记录“为什么这样做”和“不适用的场景”。
- 同一个知识点可以出现在不同入口，但尽量用链接串起来。
