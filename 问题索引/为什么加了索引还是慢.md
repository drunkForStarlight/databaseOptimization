# 为什么加了索引还是慢？

## 问题

创建索引后，SQL 仍然慢，甚至执行计划没有使用新索引。

## 常见原因

- 查询条件不匹配索引顺序。
- 联合索引没有有效利用前导列。
- 范围条件后面的列难以继续用于索引定位。
- 索引选择性太低。
- 查询返回大量数据，走索引反而成本更高。
- 需要回表的行数太多。
- 排序或分组仍然需要额外处理。
- 字段上使用函数、表达式或隐式类型转换。
- 统计信息过期，优化器成本估算错误。

## 排查路径

1. 确认 SQL 的过滤、连接、排序字段。
2. 查看已有索引和字段顺序。
3. 查看执行计划中实际使用的索引。
4. 对比扫描行数、返回行数和回表成本。
5. 检查是否存在函数、隐式转换或前导模糊匹配。
6. 必要时更新统计信息并重新观察执行计划。

## 当前结论

索引是否有效取决于查询模式、字段顺序、数据分布和优化器成本判断。加索引之前要先明确它服务哪几条 SQL，以及它能减少多少数据访问。
