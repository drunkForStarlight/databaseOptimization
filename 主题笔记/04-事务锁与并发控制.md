# 事务、锁与并发控制

很多数据库性能问题不是单条 SQL 慢，而是并发下锁等待、长事务和资源竞争造成的。

## 事务优化原则

- 事务尽量短。
- 事务内只放必须保持一致性的操作。
- 避免在事务中执行远程调用、用户交互或耗时计算。
- 按固定顺序访问资源，降低死锁概率。
- 大批量写入拆分为可控批次。

## 常见锁问题

- 长事务持有锁时间过长。
- 更新条件没有合适索引，导致锁范围扩大。
- 热点行频繁更新。
- 批量任务和在线业务争抢同一批数据。
- 不同事务访问资源顺序不一致，引发死锁。

## 诊断方向

需要收集：

- 当前活跃事务。
- 锁等待链路。
- 被阻塞 SQL 和阻塞源 SQL。
- 事务开始时间。
- 事务隔离级别。
- 涉及表和索引。

## 降低锁冲突的方法

- 为更新或删除条件建立合适索引。
- 缩短事务范围。
- 将热点计数改为异步聚合或分片计数。
- 批处理任务限速或错峰。
- 对可重试操作增加死锁重试机制。

## 隔离级别提醒

隔离级别越高，不一定性能越差，但通常会增加并发控制成本。选择隔离级别时要同时考虑一致性要求、锁行为和数据库实现差异。
