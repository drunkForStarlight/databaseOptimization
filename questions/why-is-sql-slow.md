# 为什么 SQL 会慢？

## 问题

同一条 SQL 或同一类查询为什么会出现耗时高、波动大或并发下变慢？

## 常见原因

- 扫描数据量过大。
- 没有使用合适索引。
- 执行计划选择不合理。
- 排序、分组、去重成本高。
- 连接顺序或连接方式不合适。
- 统计信息不准确。
- 锁等待或事务阻塞。
- 数据库资源瓶颈，例如 CPU、IO、内存或网络。
- 返回结果集太大。

## 排查路径

1. 看慢查询日志和实际耗时。
2. 看执行计划，确认访问路径和扫描行数。
3. 对比返回行数和扫描行数。
4. 检查过滤条件、连接条件、排序字段是否有合适索引。
5. 检查是否存在锁等待或资源瓶颈。
6. 对比数据量、参数和执行计划是否发生变化。

## 当前结论

SQL 慢通常不是单一原因。优先用执行计划和指标判断数据库到底把时间花在了哪里，再决定是改 SQL、加索引、改表结构、处理锁等待，还是调整架构。
